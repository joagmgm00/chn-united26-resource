# chn-united26-resource

## 项目简介

本仓库用于归档和发布多个独立的 HTML 页面。这些页面作为资源文件存放，不针对任何单个域名或具体网站，旨在提供一个集中、稳定的 HTML 内容分发与备份空间。

## 目录说明

```
.
├── README.md          # 本文件
├── pages/             # 存放所有 HTML 页面文件
│   ├── example1.html  # 示例页面（可删除或替换）
│   └── ...
└── assets/            # 页面引用的静态资源（CSS、JS、图片等）
    └── ...
```

- **pages/**：所有独立的 HTML 页面均放置于此目录下，文件名应反映页面内容或用途。
- **assets/**：页面所需的样式、脚本、图片等资源，建议按页面或功能归类存放。

## 页面归档说明

- 每个 HTML 页面应为一个独立、完整的文档，不依赖外部非本仓库的资源（除非特别说明）。
- 页面内容应遵循常规 HTML 规范，推荐使用 UTF-8 编码。
- 如需引用本仓库内的资源，建议使用相对路径（如 `../assets/style.css`）。
- 页面文件名建议使用小写字母、数字和连字符，避免特殊字符。

## 维护说明

- 本仓库由维护者不定期更新，新增或修改页面时会提交对应 commit。
- 欢迎通过 Issue 提出页面归档建议或报告链接失效等问题，但请注意本仓库不承诺提供任何额外服务。
- 如需引用本仓库中的页面，请直接使用 GitHub Pages 的原始文件链接，或自行部署至自己的服务器。
- 本仓库不存储任何敏感信息或违反法律法规的内容，请共同维护良好的使用环境。

## 如何使用

1. 克隆本仓库到本地：
   ```bash
   git clone https://github.com/your-username/chn-united26-resource.git
   ```
2. 在 `pages/` 下添加或修改 HTML 文件。
3. 提交并推送至 GitHub，即可通过 GitHub Pages 或直接访问原始文件查看页面。

> 注意：若需启用 GitHub Pages，请在仓库设置中将 Source 设置为 `main` 分支的 `/ (root)` 或 `/docs` 目录，具体操作请参考 GitHub 官方文档。
